## Phase 1 — v0.5 harness core (gate: the paired gate fires on an induced A/B variant diff — labeled induced — plus one honest chart, screenshotted AND queryable from D1; target 2026-09-08)
Backend Loop per unit (SPEC → SCHEMA → TEST → BUILD → VERIFY). Order matters. Scope is deliberately narrow (one judge, one judged suite, 3 targets) per the trimmed v0.5 cut — the structural suite and interactive board are v0.75/v1.
1. D1 schema + drizzle migrations (suites/suite_versions/cases/targets/judges/runs/case_results/**scores/comparisons**/aggregates/baselines/regressions/audit_log — note the **Comparison** entity carries pairwise verdicts + both transcript refs) + draft `docs/eval-harness.md` contract.
2. **Stats module FIRST**, pure functions with golden tests against hand-computed examples: bootstrap 95% CIs (resampled over cases), paired-per-case comparison (arXiv 2411.00640), rank-banding, clustered-trial treatment, win-rate-vs-anchor aggregation, **Benjamini–Hochberg FDR adjustment** (the pure function behind the nightly repeated-testing correction — see Phase 2). Property tests (CI narrows monotonically with n; paired test symmetric; BH q-values monotone in sorted p-values). **Runs inside the DO, never the 10ms Worker request path.**
3. RunOrchestrator DO: SQLite state machine, alarms for pacing/retry-with-backoff-jitter, **alarm dispatch batches capped to the 50-subrequest/invocation budget**, batched D1 writes, transcript writes to R2. **[SECURITY/Opus] BudgetLedger DO** (single instance, SQLite counters — global daily neurons against the 2,000/day share, per-visitor windows, rate limits; never KV) reserving neurons before each dispatch batch.
4. Scorer plugin interface + **llm_judge scorer, anchored pairwise** (compare candidate vs the suite's one anchor target) with position swap + k=3 sampling, Zod-parsed with bounded retry → `unscoreable`. (Structural scorer family → v0.75 with its suite.)
5. **One judged suite** (instruction-following), **3 Workers AI targets** (1 declared anchor), n_trials ≥3; runs via API/CLI only.
6. One honest chart: a **static SVG render** (custom whisker component, chart tokens — not Recharts) of 3 targets ranked with CI whiskers, every score click-through to its R2 transcript. (Interactive tie-band board → v1.)
7. **[SECURITY/Opus]** admin authoring auth (scoped bearer key for CLI publish/baseline-pin, argon2id-hashed, uniform errors); **per-key + per-IP (salted) rate limits on the admin auth + run-trigger endpoints from first deploy** (BudgetLedger DO sliding window — the posture's "hardest rate limits on auth endpoints" rule anchored here in Phase 1, not deferred to Phase 2's public-surface pass, so the deployed admin surface is throttled from first exposure); per-generation `max_tokens` on every target+judge call; BudgetLedger reservation precedes every dispatch batch.
8. **Induced A/B variant-diff demo documented and labeled induced:** one target run intact vs the same target run through a degraded prompt format (a different declared identity → a *variant diff*, not a same-identity regression), paired test fires, published as a case study that states the difference was induced to demonstrate the gate.

- **[SECURITY/Opus] render-boundary hardening (stored XSS in the viewer's browser):** raw HTML disabled in Streamdown/markdown (`javascript:`/`data:` URLs + `on*` handlers stripped, `rel="noopener noreferrer"`); every untrusted field (model_output, rationale, case input/expected) escaped at the DOM, **SVG `<text>` labels**, and table cells; **CSP** (no inline/`unsafe-*` script, scoped `frame-ancestors` for the embeddable badge) + `X-Content-Type-Options`/`Referrer-Policy`/`X-Frame-Options` on all HTML responses. Verified with a stored-`<script>`/`javascript:` payload fixture rendered on the transcript view.
